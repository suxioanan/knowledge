# Redis 缓存设计

## 缓存 Key 命名规范
格式：业务模块:实体类型:实体ID:属性
示例：order:detail:12345、user:session:abc123、product:stock:9876

## 缓存过期时间
| 数据类型 | TTL | 说明 |
|---------|-----|------|
| 用户 Session | 30分钟 | 每次请求刷新 |
| 订单详情 | 1小时 | 下单/支付时主动失效 |
| 商品库存 | 10分钟 | 高并发场景下用 Redis 扣减，异步同步 MySQL |
| 字典数据 | 24小时 | 字典修改时主动刷新 |
| 物流轨迹 | 5分钟 | 实时性要求中等 |
| 限流计数 | 滑动窗口 1分钟 | IP 级别限流 |

## 缓存更新策略
采用 Cache Aside 模式：读缓存 → 缓存未命中 → 查 DB → 写缓存
写操作：先更新 DB → 删除缓存（不更新缓存，避免双写不一致）

## 分布式锁
使用 SET NX EX 实现，key 格式：lock:业务:资源ID
锁超时时间默认 30 秒
释放锁使用 Lua 脚本校验 value，防止误删

## 缓存穿透/击穿/雪崩防护
- 穿透：布隆过滤器 + 空值缓存（TTL 1 分钟）
- 击穿：热点 Key 加互斥锁（SET NX）
- 雪崩：TTL 加随机值（±20%）
